<h2 id="案件を断れない-フリーランスの末路">「案件を断れない」フリーランスの末路</h2>
<p>フリーランスにとって、案件を断ることは<strong>収入を断ること</strong>と同義です。だから依頼が来たら受けたくなる。その気持ちは痛いほどわかります。</p>
<p>でも、キャパシティを超えて案件を受け続けると、こうなります。</p>
<ul>
<li>全ての案件で納期ギリギリ</li>
<li>クオリティが下がりクライアントの信頼を失う</li>
<li>修正対応が増え、さらに時間がなくなる</li>
<li>体調を崩して全案件がストップ</li>
<li>最悪の場合、損害賠償や違約金のリスク</li>
</ul>
<p>**これは意志の問題ではなく、仕組みの問題です。**複数案件を安全に回すための「仕組み」があれば、キャパを超える前に気づけます。</p>
<h2 id="複数案件管理の3大原則">複数案件管理の3大原則</h2>
<h3 id="原則1-全案件を1箇所で見渡せる状態を作る">原則1：全案件を1箇所で見渡せる状態を作る</h3>
<p>複数案件を管理する第一歩は、<strong>全ての案件を1つの場所に集約する</strong>ことです。</p>
<p>よくある失敗パターン:</p>
<ul>
<li>案件Aの情報はSlackに</li>
<li>案件Bの資料はメールに</li>
<li>案件Cのスケジュールはカレンダーに</li>
<li>案件Dの進捗は頭の中に</li>
</ul>
<p>これでは「今、何がどうなっているか」を把握するために、<strong>4つのツールを行き来する</strong>必要があります。</p>
<p>やるべきことはシンプルです。<strong>案件名・納期・現在のステータスだけを1箇所にまとめる</strong>。それだけで全体像が見えるようになります。</p>
<h3 id="原則2-今日やること-を3つに絞る">原則2：「今日やること」を3つに絞る</h3>
<p>複数案件を抱えていると、やるべきことが山のように見えます。でも人間が1日に集中してこなせるタスクは<strong>3〜5個</strong>です。</p>
<p>朝一番にやること:</p>
<ol>
<li>全案件のタスクを確認する</li>
<li>今日やるべきタスクを<strong>3つだけ</strong>選ぶ</li>
<li>その3つに集中する</li>
</ol>
<p>「あれもこれもやらなきゃ」と思った瞬間に、どれも中途半端になります。**3つ終わったら、次の3つ。**これだけで生産性は劇的に上がります。</p>
<h3 id="原則3-キャパシティを数値で把握する">原則3：キャパシティを数値で把握する</h3>
<p><img src="/column/images/articles/multiple-projects-management/02.jpg" alt="週間キャパシティの計算例"></p>
<!-- 画像: 週40時間に対して各案件の必要時間を積み上げたバーチャート or 表の図解 -->
<p>「なんとなく忙しい」ではなく、<strong>自分のキャパシティを数字で把握する</strong>ことが重要です。</p>
<p>計算方法はシンプル:</p>
<ul>
<li>1日の作業可能時間：8時間</li>
<li>1週間の作業可能時間：40時間</li>
<li>各案件に必要な時間を見積もる</li>
</ul>
<p>例えば:</p>
<table>
<thead>
<tr>
<th>案件</th>
<th>今週の必要時間</th>
<th>納期</th>
</tr>
</thead>
<tbody>
<tr>
<td>LPデザイン</td>
<td>15時間</td>
<td>金曜</td>
</tr>
<tr>
<td>バナー制作5枚</td>
<td>10時間</td>
<td>水曜</td>
</tr>
<tr>
<td>サイト修正</td>
<td>5時間</td>
<td>木曜</td>
</tr>
<tr>
<td>ロゴ提案</td>
<td>8時間</td>
<td>来週月曜</td>
</tr>
<tr>
<td><strong>合計</strong></td>
<td><strong>38時間</strong></td>
<td>-</td>
</tr>
</tbody>
</table>
<p>合計38時間。週40時間に対して余裕は2時間。ここに新しい案件が来たら、<strong>受けてはいけない</strong>と判断できます。</p>
<p>この判断が「感覚」ではなく「数字」で行えるのが、キャパシティ管理の価値です。</p>
<h2 id="案件のフェーズ別管理術">案件のフェーズ別管理術</h2>
<h3 id="フェーズ1-受注前-見積もり・ヒアリング段階">フェーズ1：受注前（見積もり・ヒアリング段階）</h3>
<p>案件を受ける前に確認すべきチェックリスト:</p>
<ul>
<li>納期は現実的か</li>
<li>今の案件と作業期間が被らないか</li>
<li>修正回数の上限を決めたか</li>
<li>支払い条件を確認したか</li>
</ul>
<p><strong>ここで断る勇気が、後の全てを救います。</strong></p>
<h3 id="フェーズ2-進行中-制作・確認待ち">フェーズ2：進行中（制作・確認待ち）</h3>
<p>進行中の案件で最も重要なのは、<strong>「待ち」の状態を見える化する</strong>ことです。</p>
<p>自分の手で動かせないタスク（クライアントの確認待ち、素材待ち、外注先の納品待ち）は、<strong>別のステータスで管理</strong>します。</p>
<p>待ちの案件をリストから消してしまうと、いつ送ったか忘れ、催促のタイミングを逃します。</p>
<p>ルール: <strong>3営業日返事がなければ、こちらから確認の連絡を入れる。</strong></p>
<h3 id="フェーズ3-納品後-請求・フォロー">フェーズ3：納品後（請求・フォロー）</h3>
<p>納品して終わりではありません。</p>
<ul>
<li>納品確認の返事をもらう</li>
<li>請求書を発行する（納品後3日以内）</li>
<li>入金を確認する</li>
</ul>
<p>特に<strong>請求書の発行忘れ</strong>はフリーランスの致命的なミスです。「納品済み」のステータスになったら、自動的に「請求書発行」のタスクが立つ仕組みを作りましょう。</p>
<h2 id="具体的なツール活用法">具体的なツール活用法</h2>
<p><img src="/column/images/articles/multiple-projects-management/03.jpg" alt="案件管理ツールの活用イメージ"></p>
<!-- 画像: Googleカレンダーのブロック管理例 or スプレッドシートの案件管理表のスクショ -->
<h3 id="googleカレンダーで-ブロック管理">Googleカレンダーで「ブロック管理」</h3>
<p>各案件の作業時間をカレンダーにブロックとして登録する方法です。</p>
<ul>
<li>月曜 9:00-12:00：案件A（LPデザイン）</li>
<li>月曜 13:00-17:00：案件B（バナー制作）</li>
<li>火曜 9:00-15:00：案件A（LPデザイン）</li>
</ul>
<p>こうすることで、<strong>空き時間が視覚的に分かる</strong>ようになります。空き時間がなければ、新しい案件を受けてはいけないサインです。</p>
<p><strong>メリット</strong>: 無料、すでに使っている人が多い</p>
<p><strong>デメリット</strong>: タスクの詳細管理ができない、カレンダーが埋まると見づらい</p>
<h3 id="スプレッドシートで-案件管理表">スプレッドシートで「案件管理表」</h3>
<p>全案件を一覧で管理するスプレッドシートを作る方法です。</p>
<p>管理すべき項目:</p>
<ul>
<li>案件名</li>
<li>クライアント名</li>
<li>受注日 / 納期</li>
<li>見積金額</li>
<li>ステータス（進行中/確認待ち/納品済み/請求済み/入金済み）</li>
<li>今週のToDoメモ</li>
</ul>
<p><strong>メリット</strong>: 自由にカスタマイズ可能、共有も簡単</p>
<p><strong>デメリット</strong>: 更新が手動、リマインドがない</p>
<h3 id="taskul-aiで自動管理">TASKUL（AIで自動管理）</h3>
<p><img src="/column/images/articles/multiple-projects-management/04.jpg" alt="TASKULのダッシュボード - 複数案件のタスクと納期を一元管理"></p>
<p>TASKULはフリーランス専用のAIタスク管理ツールです。</p>
<p>従来のツールとの最大の違いは、<strong>タスクの入力を自動化</strong>している点。クライアントからの依頼メッセージをそのままコピペするだけで:</p>
<ul>
<li>案件名を自動抽出</li>
<li>納期を自動特定</li>
<li>タスクを自動分解</li>
</ul>
<p>してくれます。</p>
<p>さらに毎朝LINEで「今日やるべきタスク」が届くので、朝一番に確認作業をする必要もありません。</p>
<p><strong>メリット</strong>: 入力の手間ゼロ、複数案件を自動で整理、LINEリマインド</p>
<p><strong>デメリット</strong>: チーム管理には非対応</p>
<h2 id="案件を断る技術">案件を断る技術</h2>
<p><img src="/column/images/articles/multiple-projects-management/05.jpg" alt="案件を断る判断基準のイメージ"></p>
<!-- 画像: 「断るべきサイン」のチェックリスト風図解 -->
<p>「案件管理がうまくなる」ことと同じくらい重要なのが、<strong>案件を適切に断る技術</strong>です。</p>
<h3 id="断り方のテンプレート">断り方のテンプレート</h3>
<blockquote>
<p>「ご依頼ありがとうございます。大変興味深い案件なのですが、現在進行中の案件と納期が重なっており、ご期待に沿えるクオリティでの対応が難しい状況です。○月○日以降であれば対応可能ですが、いかがでしょうか？」</p>
</blockquote>
<p>ポイントは3つ:</p>
<ol>
<li><strong>感謝を伝える</strong>（関係を壊さない）</li>
<li><strong>理由を明確にする</strong>（クオリティを担保するため）</li>
<li><strong>代替案を提示する</strong>（別の時期、もしくは知り合いを紹介）</li>
</ol>
<h3 id="断るべきサインの見極め">断るべきサインの見極め</h3>
<p>以下のサインが出たら、新しい案件を受けるべきではありません:</p>
<ul>
<li>今週のタスクが全て「急ぎ」になっている</li>
<li>睡眠時間が6時間を切っている</li>
<li>既存クライアントへの返信が24時間以上遅れている</li>
<li>「なんとかなるだろう」と思っている（なりません）</li>
</ul>
<h2 id="まとめ-パンクする前に-仕組み-を作る">まとめ：パンクする前に「仕組み」を作る</h2>
<p>複数案件を安全に回すために必要なのは、根性ではなく<strong>仕組み</strong>です。</p>
<ul>
<li>全案件を1箇所で見渡せる状態を作る</li>
<li>今日やることは3つに絞る</li>
<li>キャパシティを数字で把握する</li>
<li>「確認待ち」を可視化する</li>
<li>納品したら請求まで追跡する</li>
</ul>
<p>これらの仕組みが整えば、<strong>案件を増やしても破綻しない</strong>体制ができます。</p>
<p>「最近ちょっとキャパオーバー気味かも」と感じているなら、今日から仕組み化を始めてください。設定不要・コピペだけで始められるTASKULなら、今日中に案件管理の仕組みが手に入ります。</p>
