<h2 id="カメラマンが案件管理で抜け漏れを起こす3つの理由とは">カメラマンが案件管理で抜け漏れを起こす3つの理由とは？</h2>
<p>カメラマンの案件管理がうまくいかない原因は、<strong>現場（撮影日）に意識が集中して前後工程が見えなくなる</strong>ことにあります。撮影は華やかですが、案件全体で見ると一部の工程に過ぎません。</p>
<h3 id="理由1-香盤表は-現場用-で複数案件の管理には向かない">理由1：香盤表は「現場用」で複数案件の管理には向かない</h3>
<p>香盤表は撮影現場で必要な情報（カット内容・タイムスケジュール・モデル情報）をまとめる優れたドキュメントです。ただし、これはあくまで <strong>1案件の現場用ツール</strong>。複数案件を並行して抱えるフリーランスカメラマンには、香盤表だけでは管理しきれません。</p>
<p>「A社の撮影が終わったら、B社のセレクト、Cクライアントの請求書発行、D社のロケハン日程調整……」という状況では、香盤表より上位の管理レイヤーが必要になります。</p>
<h3 id="理由2-撮影日に意識が集中して前後工程が後回しになる">理由2：撮影日に意識が集中して前後工程が後回しになる</h3>
<p>カメラマンは撮影日が最も大事ですが、案件としては<strong>撮影前後の方が時間が長い</strong>のが実態です。</p>

<h3 id="理由3-案件ごとの納品物・請求が散らかる">理由3：案件ごとの納品物・請求が散らかる</h3>
<p>クライアントAは「JPGをギガファイルで」、クライアントBは「RAW込みでストレージ共有」、クライアントCは「USBで物理納品」という具合に、納品形式は案件ごとに異なります。</p>

<h2 id="フリーランスカメラマンが管理すべき5つの工程とは">フリーランスカメラマンが管理すべき5つの工程とは？</h2>
<p>カメラマンの案件は、<strong>1案件あたり5つの工程に分解できる</strong>と把握しておくと管理が楽になります。</p>
<p><img src="/column/images/articles/photographer-project-management/02.jpg" alt="カメラマンの案件工程を整理する様子"></p>
<h3 id="工程1-ヒアリング・見積もり">工程1：ヒアリング・見積もり</h3>
<p>依頼内容の確認、撮影目的、納品物のフォーマット、予算感を把握する工程です。ここでブレると後の工程が全て狂います。最低限ヒアリングすべき項目は以下のとおり。</p>
<ul>
<li>撮影目的（広告？SNS？社内？）</li>
<li>撮影日時の希望</li>
<li>ロケーション（屋内/屋外）</li>
<li>カット数の目安</li>
<li>納品形式・期日</li>
<li>予算感</li>
</ul>
<h3 id="工程2-事前準備-ロケハン・機材・香盤表">工程2：事前準備（ロケハン・機材・香盤表）</h3>
<p>撮影日までに準備する全タスクです。ロケハンが必要なら別日確保、機材レンタルの予約、当日のアシスタント手配、クライアントへの最終確認など、<strong>撮影日の3〜7日前から動く</strong>のが安全です。</p>
<h3 id="工程3-撮影当日">工程3：撮影当日</h3>
<p>香盤表に沿って撮影を進めます。<strong>ここは香盤表が主役</strong>になる工程。撮影中の追加要望や変更があれば、すぐに案件ノートにメモします（後で「言った言わない」を防ぐため）。</p>
<h3 id="工程4-セレクト・レタッチ">工程4：セレクト・レタッチ</h3>
<p>撮影データから納品候補を選び、レタッチを行います。カット数が多い案件ではこの工程に時間がかかるため、<strong>セレクト締切日を案件管理ツールに必ず登録</strong>します。</p>
<h3 id="工程5-納品・請求">工程5：納品・請求</h3>
<p>納品データの送付、請求書発行、入金確認まで完了して案件クローズです。**「納品=完了」ではなく「入金確認=完了」**と定義すると請求漏れがなくなります。</p>
<h2 id="カメラマンが案件管理を一元化する3ステップとは">カメラマンが案件管理を一元化する3ステップとは？</h2>
<p>案件管理を一元化する具体的な手順は、<strong>「案件ボード化→工程分解→納品請求セット登録」の3ステップ</strong>です。</p>
<p><img src="/column/images/articles/photographer-project-management/03.jpg" alt="3ステップで案件管理を整える図"></p>
<h3 id="ステップ1-案件ボードで全案件を見える化する">ステップ1：案件ボードで全案件を見える化する</h3>
<p>進行中の全案件を1画面で見渡せる「案件ボード」を作ります。Trelloのカンバン、Notionのデータベース、TASKULの案件一覧など、形式は何でもOK。</p>
<p>ボードの列は以下が基本です。</p>
<ul>
<li>ヒアリング中</li>
<li>撮影前準備</li>
<li>撮影完了（セレクト中）</li>
<li>納品済み（請求待ち）</li>
<li>入金完了（クローズ）</li>
</ul>
<p>各案件カードに、クライアント名・撮影日・納品予定日・金額を記載すると、<strong>頭の中で覚えなくていい状態</strong>になります。</p>
<h3 id="ステップ2-工程ごとにタスクを分解する">ステップ2：工程ごとにタスクを分解する</h3>
<p>各案件カードの中に、5工程それぞれのタスクを分解して登録します。例えば「A社 製品撮影」案件なら：</p>
<ul class="contains-task-list">
<li class="task-list-item"><input type="checkbox" disabled> ヒアリング電話（5/10）</li>
<li class="task-list-item"><input type="checkbox" disabled> 見積もり提出（5/12）</li>
<li class="task-list-item"><input type="checkbox" disabled> ロケハン（5/15）</li>
<li class="task-list-item"><input type="checkbox" disabled> 機材予約（5/18）</li>
<li class="task-list-item"><input type="checkbox" disabled> 撮影当日（5/20）</li>
<li class="task-list-item"><input type="checkbox" disabled> セレクト送付（5/22）</li>
<li class="task-list-item"><input type="checkbox" disabled> レタッチ完了（5/25）</li>
<li class="task-list-item"><input type="checkbox" disabled> 納品（5/26）</li>
<li class="task-list-item"><input type="checkbox" disabled> 請求書発行（5/27）</li>
<li class="task-list-item"><input type="checkbox" disabled> 入金確認（6/30）</li>
</ul>
<p>このレベルまで分解すると、<strong>「次に何をするか」が常に明確</strong>になります。</p>
<h3 id="ステップ3-納品予定日と請求日を必ずセットで登録">ステップ3：納品予定日と請求日を必ずセットで登録</h3>
<p>最重要のルールです。<strong>案件登録時に「納品予定日」と「請求書送付日」を必ずセットで登録</strong>します。</p>
<p>カメラマンが請求漏れを起こす最大の原因は、納品で完了した気になって請求書発行を忘れること。納品日と請求日をセットで管理すれば、月末の請求漏れがほぼゼロになります。</p>

<h2 id="カメラマンの案件管理でよくある失敗とは">カメラマンの案件管理でよくある失敗とは？</h2>
<p>実際に多くのフリーランスカメラマンが陥る失敗パターンを3つ紹介します。</p>
<p><img src="/column/images/articles/photographer-project-management/05.jpg" alt="カメラマンの案件管理での失敗例"></p>
<h3 id="失敗1-撮影日だけスケジューラーに入れて他工程を忘れる">失敗1：撮影日だけスケジューラーに入れて他工程を忘れる</h3>
<p>最も多い失敗です。Googleカレンダーに「5/20 A社撮影」とだけ入れて、ロケハン・セレクト・納品の予定が空欄。撮影直前に「あれ、ロケハンしてない」と慌てる、撮影後に「セレクトいつまでだっけ」と探す、という事態が起きます。</p>
<p><strong>対策</strong>：案件登録時に5工程の全タスクを同時に作る。</p>
<h3 id="失敗2-納品後の請求漏れ">失敗2：納品後の請求漏れ</h3>
<p>納品で安心して請求書発行を忘れるパターン。月末に気づくと、入金が翌月にずれて資金繰りに影響します。</p>
<p><strong>対策</strong>：納品日と請求書発行日を必ずセットで登録。納品タスクを完了する前に「請求書発行タスク」を生成するワークフローにする。</p>
<h3 id="失敗3-クライアントとのやり取り履歴が散らばる">失敗3：クライアントとのやり取り履歴が散らばる</h3>
<p>メール、LINE、Messengerなど、クライアントごとに連絡手段がバラバラで、撮影内容の変更履歴がどこに書いたか分からなくなる。**「あの追加カットのリクエスト、どのメッセージで来たっけ」**を探す時間が無駄に積み上がります。</p>
<p><strong>対策</strong>：案件管理ツールの案件ノートに、外部のやり取り（メール本文、LINE スクショ）を集約する。一元化すれば後から検索できる。</p>
