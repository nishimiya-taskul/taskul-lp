<h2 id="イラストレーターの案件管理が難しい理由">イラストレーターの案件管理が難しい理由</h2>
<p>イラストレーターの仕事は、他のフリーランスと比べて<strong>案件管理の難易度が高い</strong>という特徴があります。</p>
<p>なぜか。それは1つの案件の中に<strong>複数の工程</strong>があり、しかも<strong>クライアントの確認待ち</strong>が頻繁に発生するからです。</p>
<h3 id="一般的なイラスト案件の工程">一般的なイラスト案件の工程</h3>
<p><img src="/column/images/articles/illustrator-project-management/02.png" alt="イラスト案件の工程フロー"></p>
<!-- 画像未設定: ヒアリング→ラフ→確認待ち→線画→着色→納品→請求の工程フロー図 -->
<ol>
<li>ヒアリング・要件確認</li>
<li>ラフ案作成（複数案の場合も）</li>
<li><strong>クライアント確認待ち</strong></li>
<li>ラフ修正</li>
<li>線画作成</li>
<li><strong>クライアント確認待ち</strong></li>
<li>着色</li>
<li><strong>クライアント確認待ち</strong></li>
<li>最終修正・納品</li>
<li>請求書発行</li>
</ol>
<p>1案件でこれだけの工程があるのに、フリーランスのイラストレーターは<strong>常時3〜5案件を並行</strong>して進めています。</p>
<p>つまり、頭の中で管理しなければならないのは:</p>
<ul>
<li>案件A：ラフ修正中（明日締切）</li>
<li>案件B：クライアント確認待ち（3日前に送った）</li>
<li>案件C：線画作成中（来週月曜締切）</li>
<li>案件D：ヒアリング前（木曜打ち合わせ）</li>
<li>案件E：請求書まだ送ってない</li>
</ul>
<p><strong>これを頭の中だけで管理するのは、いつか必ず破綻します。</strong></p>
<h2 id="イラストレーターが案件管理で失敗する3つのパターン">イラストレーターが案件管理で失敗する3つのパターン</h2>
<h3 id="パターン1-確認待ちの案件を忘れる">パターン1：確認待ちの案件を忘れる</h3>
<p>「先週ラフを送ったけど、返事が来ない。催促すべき？ いつ送ったっけ…」</p>
<p>クライアントからの返信待ちは、<strong>自分のタスクリストから消えがち</strong>です。でも、返事が来ないまま放置すると、納期が押して最終的に自分が徹夜する羽目になります。</p>
<h3 id="パターン2-リテイク回数を把握していない">パターン2：リテイク回数を把握していない</h3>
<p>「このイラスト、もう何回修正したっけ…？」</p>
<p>契約時に「修正2回まで」と決めていても、回数を記録していないと、ずるずると無限修正に巻き込まれます。3回目以降は追加料金を請求すべきなのに、回数が曖昧だと言い出せません。</p>
<h3 id="パターン3-請求漏れ">パターン3：請求漏れ</h3>
<p>納品したのに請求書を送り忘れる。これは笑い事ではなく、<strong>フリーランスの収入を直接的に減らす致命的なミス</strong>です。</p>
<p>特に月末に複数の納品が重なると、「あの案件、請求書送ったっけ？」が頻発します。</p>
<h2 id="イラストレーターに必要な案件管理の仕組み">イラストレーターに必要な案件管理の仕組み</h2>
<p><img src="/column/images/articles/illustrator-project-management/03.png" alt="案件管理に必要な3要素の図解"></p>
<!-- 画像未設定: 「工程ステータス管理」「リテイク回数記録」「納品→請求チェック」の3要素図 -->
<p>上記の失敗を防ぐために、イラストレーターの案件管理には<strong>最低限3つの要素</strong>が必要です。</p>
<h3 id="要素1-工程ごとのステータス管理">要素1：工程ごとのステータス管理</h3>
<p>「この案件は今どの段階にあるか」が一目でわかる必要があります。</p>
<ul>
<li>ヒアリング中</li>
<li>ラフ作成中</li>
<li><strong>確認待ち</strong>（ここが重要！）</li>
<li>線画作成中</li>
<li>着色中</li>
<li>納品済み</li>
<li>請求済み</li>
</ul>
<p>特に**「確認待ち」を独立したステータスにする**ことが重要です。自分のタスクではないけれど、放置してはいけない状態を可視化できます。</p>
<h3 id="要素2-リテイク回数の記録">要素2：リテイク回数の記録</h3>
<p>案件ごとに「何回修正したか」を記録する欄があれば、追加料金の請求判断が明確になります。</p>
<ul>
<li>ラフ修正：2回（契約内）</li>
<li>線画修正：1回（契約内）</li>
<li>着色修正：3回目 → <strong>追加料金対象</strong></li>
</ul>
<h3 id="要素3-納品-請求のチェックリスト">要素3：納品→請求のチェックリスト</h3>
<p>納品したら自動的に「請求書を送る」というタスクが発生する仕組みがあれば、請求漏れを防げます。</p>
<h2 id="案件管理ツールの選び方-イラストレーター視点">案件管理ツールの選び方：イラストレーター視点</h2>
<p>世の中に案件管理ツールはたくさんありますが、イラストレーターが選ぶ際のポイントは以下の3つです。</p>
<h3 id="ポイント1-工程管理ができるか">ポイント1：工程管理ができるか</h3>
<p>BacklogやJiraは高機能ですが、チーム開発向けに設計されています。イラストレーターが1人で使うには重すぎます。</p>
<p>必要なのは「ラフ→線画→着色→納品→請求」という<strong>自分の工程だけ</strong>を管理できるシンプルさです。</p>
<h3 id="ポイント2-確認待ちを管理できるか">ポイント2：確認待ちを管理できるか</h3>
<p>多くのタスク管理ツールは「自分がやるタスク」の管理には強いですが、<strong>「相手待ちの状態」を管理する機能</strong>は弱い。</p>
<p>カンバンボードで「確認待ち」のレーンを作れるツール、もしくは自動リマインド機能があるツールを選びましょう。</p>
<h3 id="ポイント3-見積書・請求書との連動">ポイント3：見積書・請求書との連動</h3>
<p>案件管理と請求管理が別々のツールだと、どうしても「納品したけど請求を忘れた」が発生します。</p>
<p>理想は、案件のステータスを「納品済み」にしたら<strong>自動で請求タスクが立つ</strong>仕組みです。</p>
<h2 id="おすすめの管理方法">おすすめの管理方法</h2>
<h3 id="方法1-googleスプレッドシート-無料で始めたい人向け">方法1：Googleスプレッドシート（無料で始めたい人向け）</h3>
<p>最もシンプルな方法は、Googleスプレッドシートで案件管理表を作ることです。</p>
<p>列の構成例:</p>
<ul>
<li>案件名</li>
<li>クライアント名</li>
<li>受注日</li>
<li>納期</li>
<li>現在のステータス</li>
<li>リテイク回数（ラフ/線画/着色）</li>
<li>請求ステータス</li>
<li>備考</li>
</ul>
<p><strong>メリット</strong>: 無料、カスタマイズ自由、スマホからも見れる</p>
<p><strong>デメリット</strong>: リマインド機能がない、入力が手動、見た目が地味でモチベーションに繋がりにくい</p>
<h3 id="方法2-trello-視覚的に管理したい人向け">方法2：Trello（視覚的に管理したい人向け）</h3>
<p>Trelloのカンバンボードを使って、案件の進捗を視覚的に管理する方法です。</p>
<p>レーンの構成例:</p>
<ul>
<li>ヒアリング中</li>
<li>ラフ作成中</li>
<li>確認待ち</li>
<li>線画〜着色中</li>
<li>納品済み</li>
<li>請求済み</li>
</ul>
<p>カードを移動するだけで進捗が更新されるので、直感的に使えます。</p>
<p><strong>メリット</strong>: 視覚的で分かりやすい、無料で十分使える</p>
<p><strong>デメリット</strong>: リテイク回数の管理が面倒、案件が増えるとカードが溢れる</p>
<h3 id="方法3-taskul-aiで自動化したい人向け">方法3：TASKUL（AIで自動化したい人向け）</h3>
<p><img src="/column/images/articles/illustrator-project-management/04.png" alt="TASKULでイラスト案件を管理している画面"></p>
<!-- 画像未設定: TASKULの画面キャプチャ（イラスト案件のタスクが並んでいる状態） -->
<p>TASKULはフリーランス・クリエイター専用のAIタスク管理ツールです。</p>
<p>クライアントからの依頼メッセージを<strong>そのままコピペするだけ</strong>で、AIが自動的に:</p>
<ul>
<li>案件名を抽出</li>
<li>期日を特定</li>
<li>タスクに分解</li>
</ul>
<p>してくれます。</p>
<p><strong>メリット</strong>: 入力の手間がほぼゼロ、毎朝LINEで今日のタスクが届く、請求書もワンタップ</p>
<p><strong>デメリット</strong>: チーム利用には非対応</p>
<h2 id="案件管理を始めるための3ステップ">案件管理を始めるための3ステップ</h2>
<p>「ツールは決まった。でも何から始めればいい？」という方のために、今日からできる3ステップを紹介します。</p>
<h3 id="ステップ1-今抱えている案件を全部書き出す">ステップ1：今抱えている案件を全部書き出す</h3>
<p>まず、今進行中の案件を全てリストアップします。頭の中にあるものを全部外に出してください。</p>
<ul>
<li>案件名</li>
<li>納期</li>
<li>今どの工程にあるか</li>
</ul>
<p>これだけでOKです。完璧に書く必要はありません。</p>
<h3 id="ステップ2-確認待ち-の案件にフラグを立てる">ステップ2：「確認待ち」の案件にフラグを立てる</h3>
<p>リストの中から、<strong>クライアントの返答待ちになっている案件</strong>を特定します。そして「いつ送ったか」「いつまでに返事がなければ催促するか」を決めます。</p>
<p>この1つのアクションだけで、「確認待ちの案件を忘れる」というミスがなくなります。</p>
<h3 id="ステップ3-週に1回15分の振り返りをする">ステップ3：週に1回15分の振り返りをする</h3>
<p>毎週金曜日の夕方に15分だけ:</p>
<ul>
<li>各案件のステータス更新</li>
<li>来週の納期確認</li>
<li>請求漏れがないかチェック</li>
</ul>
<p>これを習慣にするだけで、案件管理は回り始めます。</p>
<h2 id="まとめ-イラストレーターの案件管理は-確認待ち-がカギ">まとめ：イラストレーターの案件管理は「確認待ち」がカギ</h2>
<p>イラストレーターの案件管理で最も重要なのは、<strong>「確認待ち」という自分のタスクではない状態を可視化すること</strong>です。</p>
<ul>
<li>工程ごとのステータスを明確にする</li>
<li>リテイク回数を記録して追加料金の判断根拠にする</li>
<li>納品→請求の流れを仕組み化する</li>
</ul>
<p>複数案件を安定して回せるようになると、<strong>受注できる案件数が増え、結果的に収入アップ</strong>につながります。</p>
<p>まだ案件管理の仕組みがないなら、今日から始めてください。ツール選びで迷うなら、設定不要でコピペだけで始められるTASKULがおすすめです。</p>
