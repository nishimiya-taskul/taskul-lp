22:T2a6e,<p>GooglebotはHTMLファイルの<strong>最初の2MB（約2,097,152バイト）しか取得しません</strong>。2MBを超えた部分は、クロールもレンダリングもされず「存在しない扱い」です。2026年3月、Google公式ブログでこの制限が改めて明言されました。</p>
<p>「記事を書いたのに順位が上がらない」「構造化データを入れたのにリッチリザルトが出ない」――その原因がHTMLサイズにある可能性があります。特にWordPressでプラグインを10個以上使っているサイトや、ページビルダーで構築したサイトは要注意です。</p>
<p>この記事では、HTML 2MB制限の具体的な影響と、HTMLサイズの確認方法、今すぐできる5つの対策を解説します。</p>
<h2 id="html-2mb制限とは-googleはページのどこまで読んでいる">HTML 2MB制限とは？Googleはページのどこまで読んでいる？</h2>
<p>Googlebotがページをクロールする際、HTMLファイルの**最初の2MB（約2,097,152バイト）**までしか取得しません。</p>
<p>これはHTMLの「ファイルサイズ」の話です。画像やCSS、JSの外部ファイルは含まれません。あくまでHTMLドキュメント本体のサイズです。</p>
<h3 id="htmlが2mbを超えるとどうなる">HTMLが2MBを超えるとどうなる？</h3>
<ul>
<li>2MBより後ろにあるコンテンツは<strong>クロールされない</strong></li>
<li>レンダリングもされないため、<strong>Googleのインデックスに含まれない</strong></li>
<li>構造化データが後半にある場合、<strong>リッチリザルトに反映されない</strong></li>
<li>内部リンクが後半にある場合、<strong>リンクジュースが渡らない</strong></li>
</ul>
<p>つまり、ページの後半に重要なコンテンツを置いている場合、それはGoogleにとって「存在しないもの」です。</p>
<h2 id="htmlが2mbを超えるのはどんなケース-よくある事故パターン">HTMLが2MBを超えるのはどんなケース？よくある事故パターン</h2>
<h3 id="1-ナビゲーションや装飾でhtmlが肥大化">1. ナビゲーションや装飾でHTMLが肥大化</h3>
<p>ヘッダーやフッター、サイドバーに大量のHTMLを使っているサイトは要注意です。本文が始まる前にHTMLの大部分を消費してしまい、肝心の記事コンテンツがGoogleに読まれない可能性があります。</p>
<p>特にWordPressでプラグインを10個以上入れているサイトは、知らない間にHTMLが膨らんでいることがあります。</p>
<h3 id="2-インラインcss・jsの埋め込み">2. インラインCSS・JSの埋め込み</h3>
<p>CSSやJavaScriptをHTMLに直接書き込む（インライン化する）と、HTMLサイズが急激に増えます。特にメール配信用のHTMLをそのままWebに流用しているケースで起きがちです。</p>
<h3 id="3-base64エンコードされた画像">3. base64エンコードされた画像</h3>
<p>画像をbase64でHTMLに埋め込むと、1枚の画像で数十KB〜数百KB消費します。小さなアイコンなら問題ありませんが、大きな画像をbase64で複数埋め込むとHTMLサイズが一気に膨らみます。</p>
<h3 id="4-spaでのコンテンツ遅延読み込み">4. SPAでのコンテンツ遅延読み込み</h3>
<p>React、Vue、AngularなどのSPA（Single Page Application）で、JavaScriptによってコンテンツを後から読み込む場合、<strong>JSも取得された2MBの範囲内でしか実行されません</strong>。</p>
<p>画面に表示されていても、Googlebot的には存在しない——これがSPAのSEOリスクです。</p>
<h3 id="5-構造化データがフッターにある">5. 構造化データがフッターにある</h3>
<p>JSON-LDの構造化データをフッター付近に配置しているサイトは多いですが、HTMLが大きいサイトでは2MBの制限によって構造化データが読み取られない可能性があります。</p>
<h2 id="自分のサイトのhtmlサイズはどう確認する">自分のサイトのHTMLサイズはどう確認する？</h2>
<h3 id="htmlサイズの確認手順">HTMLサイズの確認手順</h3>
<ol>
<li>ブラウザでページを開く</li>
<li>右クリック →「ページのソースを表示」</li>
<li>表示されたHTMLを全選択してテキストエディタにコピー</li>
<li>ファイルサイズを確認</li>
</ol>
<p>または、ターミナルで以下を実行：</p>
<pre><code>curl -s https://あなたのサイト.com/ | wc -c
</code></pre>
<p><strong>目安：500KB以下なら安全。1MB超えたら要注意。1.5MB超えたら対策必須。</strong></p>
<h3 id="google-search-consoleでの確認">Google Search Consoleでの確認</h3>
<p>Google Search Consoleの「URL検査」ツールで、ページのHTMLがどこまで取得されているか確認できます。「クロール済みのページを表示」でGooglebotが実際に取得したHTMLを見ることができます。</p>
<h2 id="html-2mb制限への対策は-今すぐできる5つの方法">HTML 2MB制限への対策は？今すぐできる5つの方法</h2>
<h3 id="1-重要なものは-htmlの上部-に置く">1. 重要なものは「HTMLの上部」に置く</h3>
<pre><code class="language-html">&#x3C;head>
  &#x3C;!-- ここに重要なものを集中 -->
  &#x3C;title>ページタイトル&#x3C;/title>
  &#x3C;meta name="description" content="...">
  &#x3C;link rel="canonical" href="...">
  &#x3C;script type="application/ld+json">構造化データ&#x3C;/script>
&#x3C;/head>
</code></pre>
<p>title、meta description、canonical、構造化データは<strong>必ず<code>&#x3C;head></code>内に配置</strong>してください。bodyの後半に置いてはいけません。</p>

<h3 id="3-ssr・ssgを採用する">3. SSR・SSGを採用する</h3>
<p>SPAの場合、**サーバーサイドレンダリング（SSR）<strong>または</strong>静的サイト生成（SSG）**に切り替えることで、初期HTMLにコンテンツを含めることができます。</p>
<ul>
<li><strong>Next.js</strong>: SSR / SSG両対応</li>
<li><strong>Nuxt.js</strong>: Vue向けのSSR / SSG</li>
<li><strong>Astro</strong>: 静的サイト生成に特化</li>
</ul>
<p>フリーランスのポートフォリオサイトやLP制作で、見た目は動的だけど実はSSGで生成している、というのがSEO的にはベストです。</p>
<h3 id="4-遅延読み込みを適切に使う">4. 遅延読み込みを適切に使う</h3>
<p>画像の遅延読み込み（lazy loading）はHTMLサイズに影響しませんが、コンテンツの遅延読み込みはSEOリスクがあります。</p>
<ul>
<li><strong>OK</strong>: <code>&#x3C;img loading="lazy"></code> — HTMLには<code>&#x3C;img></code>タグが最初からある</li>
<li><strong>NG</strong>: JSで後からDOMを生成してコンテンツを表示 — HTMLに含まれない</li>
</ul>
<h2 id="フリーランスのサイトで特に気をつけるべきことは">フリーランスのサイトで特に気をつけるべきことは？</h2>
<p>フリーランスのポートフォリオサイトやブログで注意すべきは以下です：</p>
<ul>
<li><strong>WordPressのプラグイン過多</strong>: 各プラグインがHTMLにコードを挿入する。不要なプラグインを削除</li>
<li><strong>ページビルダーの出力</strong>: Elementor等のページビルダーは大量のHTMLを生成する</li>
<li><strong>アニメーションライブラリ</strong>: HTMLに大量のインラインスタイルを追加するものがある</li>
<li><strong>埋め込みウィジェット</strong>: SNSの埋め込み、地図、カレンダー等はiframeを使い、HTMLサイズへの影響を抑える</li>
</ul>
